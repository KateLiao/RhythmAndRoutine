# 首页洞察卡片高度与生成态优化

> 父任务：`07-09-homepage-insight-cards`  
> 产品文档：`docs/v0.3.0 版本需求/新增特性-首页右侧建议卡片.md`  
> 类型：轻量 UX 优化（PRD-only）

## 目标

让首页右侧三张洞察卡在桌面布局下与左侧日历高度对齐，并通过「默认精简 + 交互展开」控制信息密度；同时让手动「更新」生成过程可感知，失败可恢复，避免长时间无反馈的 loading。

## 背景

- 左侧日历有 `max-height: calc(100vh - 11.5rem)`，右侧三卡内容撑开，视觉不齐。
- 手动刷新目前仅旋转图标；LLM 同步请求耗时长时用户焦虑；失败只靠顶部 notice（3 秒消失），卡片本身无错误态。

## 已确认需求

### 1. 高度对齐与信息折叠

- 桌面宽屏：右侧 `insight-column` 总高度与左侧日历壳对齐（同列 stretch / 同 max-height）。
- 三卡在列内均分可用高度；内容过多时卡内滚动，不把整列撑破。
- **默认面上只保留主信息**；次要信息通过「展开详情」交互呈现：

| 卡片 | 默认可见 | 默认折叠（可展开） |
|------|----------|-------------------|
| 此刻建议 | 标题区、headline、judgment、推荐下一步、操作按钮 | 一句话原因 `reason` |
| 节奏发现 | 标题区、发现陈述、操作入口 | 证据 `evidence`、影响 `impact` |
| 本周轨道 | 标题区、状态、负荷条、summary | 调整建议 `suggestion` |

- 验收文档要求的「发现 + 证据」等仍须可访问，只是默认不占满首屏。
- 窄屏（三卡横排/竖排）不强制与日历等高，但折叠行为保持一致。

### 2. 生成态可感知

- 点击「更新」后：**先清除卡片正文**；按钮禁用 + 旋转；**卡片内显示生成状态文案**（如「正在重新生成…」）。
- 超过约 8 秒仍未返回：文案升级为「仍在生成，请稍候…」，降低焦虑。
- 成功：清除生成态，更新内容，保留现有顶部成功 notice。
- 失败：卡片内展示错误文案 +「重试」；顶部 notice 同步失败信息；错误态直到用户重试或成功刷新才清除。
- 客户端超时（约 50s）：按失败处理，提示超时可重试，避免无限 loading。
- 服务端单次 LLM 约 35s 超时后降级规则；手动刷新允许 rules 覆盖旧 AI 快照。
- 浏览器控制台与服务端日志输出 `[home-insights]` / `[llm]` 前缀，便于排查超时/降级原因。
- 本轮不做异步 job / 流式进度；仍为同步 regenerate，只补齐前端可感知层。

### 3. 装饰与横向溢出

- 此刻建议顶部同心圆为底层绝对定位装饰，不单独占位。
- 洞察卡禁止左右滚动。

## 非目标

- 不改刷新策略（定时 / 手动 / GET 只读）与 LLM prompt。
- 不做 Suggestion 实体（Phase C）。
- 不做服务端流式进度条。

## 验收标准

- [x] 桌面首页：右侧三卡总高度与左侧日历视觉对齐，不显著溢出或留白断层
- [x] 默认折叠次要字段；展开后可见完整信息；再次收起恢复精简
- [x] 点击更新后卡片正文先清空，并有明确生成中文案
- [x] 长时间等待有二次提示；超时或失败有卡片内错误 + 可重试；有排查日志
- [x] 成功后生成态消失，内容更新
- [x] 顶部纹理不占位；无左右滚动条
- [x] 产品需求文档已同步本节 UX 约定

## Out of Scope

- Cron / 冷启动生成的服务端进度推送
- 改动 Facts / 快照 schema
