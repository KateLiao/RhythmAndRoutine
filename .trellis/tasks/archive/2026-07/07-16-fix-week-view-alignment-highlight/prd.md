# 修复周视图列对齐与日期高亮

## Goal

让周视图的表头与时间网格保持严格对齐，并让“今天”和“当前锚点日期”的视觉语义清晰、不产生两列都像被选中的误解。

## Background

- 周视图表头与时间网格位于两个独立布局容器中；时间网格出现纵向滚动条后，两者可用宽度不同，列分隔线逐列偏移。
- `anchorDate` 当前通过 `.is-selected` 给表头和整列添加背景；`todayKey` 通过 `.is-today` 给时间网格整列添加背景，但表头只改变日期数字颜色，因此同一状态在表头和内容区的表达不一致。
- 相关实现位于 `src/components/calendar/week-timeline.tsx:80` 与 `src/app/globals.css:397`。

## Requirements

- 表头与时间网格必须由同一个滚动容器控制水平宽度和滚动位置。
- 只有“今天”可以使用贯穿表头与时间网格的淡色整列背景。
- 当前锚点日期保留轻量、非整列的日期标识；当锚点日期与今天重合时，两种状态应自然合并。
- 保持现有周视图日程渲染、纵向自动定位、拖拽改期和小屏最小列宽行为不变。

## Acceptance Criteria

- [x] 周视图出现纵向滚动条时，表头与七个日期列的分隔线仍逐列对齐。
- [x] 横向滚动周视图时，表头和时间网格同步移动。
- [x] 今天的表头和时间网格使用同一套淡色整列背景。
- [x] 非今天的锚点日期不再给表头或时间网格整列铺底色，而是使用日期下划线等轻量标识。
- [x] 时间刻度始终位于网格左侧，七天日程网格与刻度处于同一行；09:00 日程不得落到刻度区域下方。
- [x] 周视图原有日程块、时间刻度和当前时间线功能正常。
- [x] 项目 lint、类型检查及相关测试通过。

## Out of Scope

- 不改变日视图、月视图的日期高亮设计。
- 不改变 `anchorDate` 的状态模型、周导航逻辑或日程数据契约。
