Agent 执行过程交互原型

这个原型把真实工具调用包装成用户可理解的“处理过程”，并放在最终回复气泡之前。过程区域用于表达 Agent 正在处理什么，最终回复承接结果并给出结论。

轻量过程提示 默认摘要 点击展开详情 自然语言追问

交互重点

  • 执行过程位于最终回复之前，但视觉权重低于最终回复。
  • 统一收拢为“处理过程”，避免像开发调试日志或回答后的附录。
  • 步骤标题使用“检查 / 回顾 / 参考 / 等待确认”等用户化表达。
  • 每一步默认只显示一句摘要，点击后再展示检查范围、结果和简短判断。

可直接给 Coding Agent 的方向

  • 不要改变真实 Tool Call，只改展示层的包装与映射。
  • 普通用户不展示 tool name、参数、JSON、数据库字段。
  • 当信息不足时，主回复中自然语言追问，过程区域只辅助标记“等待确认”。
