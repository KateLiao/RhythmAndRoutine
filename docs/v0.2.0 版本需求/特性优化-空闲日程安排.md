# 空闲日程安排（个人日程占位）

> 版本：v0.2.0  
> 状态：已澄清，可进入开发  
> 更新日期：2026-06-25

## 1. 背景与问题

当前「今日安排」日视图中，双击空白时间段或点击「添加日程」时，只能创建**关联目标/任务**的日程块。这与普通日历（Google Calendar、飞书日历等）的使用习惯不一致——用户常常需要先把**外部占用**或**生活事务**标在日历上，用于：

1. **时间占位**：会议、通勤、午休、约会等，占用时段后避免误排高专注任务。
2. **视觉留白**：让日视图更贴近真实的一天，而不是只有「目标推进」相关安排。
3. **低摩擦记录**：不需要为此创建目标或任务，也不应影响目标/Routine 的统计口径。

因此需要引入**个人日程**（亦称空闲日程占位）：独立于目标体系的日历事件，行为类似通用日历中的「个人事件」。

## 2. 设计目标

1. 在今日安排页添加日程时，用户可**选择类型**：目标日程 vs 个人日程。
2. 个人日程**不强制关联**目标、任务或 Routine。
3. 个人日程在日/周/月视图中**可见、可拖动、可改期、可删除**，样式与任务/Routine 区分。
4. 个人日程**不计入**目标周投入、任务执行历史、Routine 完成率等业务统计。
5. 后端复用现有 `ScheduleBlock` 模型（`goalId`/`taskId`/`routineId` 均为空），不新增表。

## 3. 用户场景

| 场景 | 期望行为 |
|------|----------|
| 双击日视图空白格 | 弹出类型选择：「个人日程」或「目标日程」 |
| 选择个人日程 | 打开简化表单：名称、日期、起止时间 |
| 选择目标日程 | 打开现有「安排到日历」表单（关联目标/任务） |
| 从目标/任务详情「安排到日历」 | 直接进入目标日程流程，**不**出现类型选择 |
| 查看日视图 | 个人日程以中性色块展示，标签为「个人」 |
| 双击已有个人日程 | 打开编辑弹窗，可改名称/时间或删除 |
| 记录执行反馈 | 个人日程**不展示**「记录执行」入口（非目标推进事项） |

## 4. 交互设计

### 4.1 添加入口与类型选择

**触发入口（需类型选择）：**

- 日视图时间轴空白处双击
- 日视图底部「添加日程」
- 日视图空状态「安排一件事」

**不触发类型选择（保持原逻辑）：**

- 目标详情页「安排到日历」
- 任务详情页「安排到日历」
- Routine 相关安排入口

**类型选择弹窗：**

```
┌─────────────────────────────────────┐
│  添加日程                            │
│  这一天想标记什么？                   │
├─────────────────────────────────────┤
│  [ 个人日程 ]                        │
│  会议、通勤、休息等占位，不关联目标    │
├─────────────────────────────────────┤
│  [ 目标日程 ]                        │
│  推进某个目标或任务的具体安排          │
└─────────────────────────────────────┘
```

选择后保留已预填的日期与时间段（来自双击位置或当前选中日期）。

### 4.2 个人日程创建表单

字段：

| 字段 | 必填 | 说明 |
|------|------|------|
| 日程名称 | 是 | 如「团队周会」「午休」「通勤」 |
| 日期 | 是 | 默认当前选中日期 |
| 开始时间 | 是 | 双击时按点击位置预填，15 分钟对齐 |
| 结束时间 | 是 | 默认开始时间 + 1 小时 |

无目标、任务、Routine、精力等级等字段。

### 4.3 日历展示

- **类型标签**：`个人`（英文角标 `PERSONAL`）
- **视觉**：中性蓝灰色调，实线边框（与 Routine 虚线、任务紫色区分）
- **元信息**：不显示关联目标；可显示时长
- **操作**：支持拖动改时、边缘拉伸、双击编辑、删除；**无**「记录执行」按钮

### 4.4 编辑与删除

复用现有「移动或编辑日程」弹窗逻辑：

- 关联目标默认为「不关联目标」
- 不展示任务多选（无关联目标时）
- 支持原地改期（`moveInPlace`）与删除

## 5. 数据模型

### 5.1 前端 `ScheduleItem`

```ts
type ScheduleItem = {
  // ...现有字段
  kind: "task" | "routine" | "review" | "personal"
  goalId: string  // 个人日程为空字符串
}
```

**kind 判定规则：**

| 条件 | kind |
|------|------|
| 有 `routineId` | `routine` |
| 无 `goalId` 且无 `taskId` 且无 `routineId` | `personal` |
| 其他 | `task` |

### 5.2 后端

复用 `ScheduleBlock`：

- `goalId`: `null`
- `taskId`: `null`
- `routineId`: `null`
- `source`: `manual`
- `title`: 用户输入

创建接口 `POST /api/schedule` 已支持可选 `goalId`，无需 schema 变更。

## 6. 统计与 Agent 边界

| 维度 | 个人日程是否计入 |
|------|------------------|
| 今日安排块数量（日视图摘要） | 是 |
| 今日完成环（completed/total） | 是（若用户手动标记完成则计入，默认不主动引导反馈） |
| 目标周投入分钟 | 否 |
| 任务执行历史 / 任务状态聚合 | 否 |
| Routine 完成率 | 否 |
| 回顾生成中的目标推进分析 | 否（可出现在「当日时间占用」描述，但不作为任务完成） |
| Agent ChangeSet 创建个人日程 | 是（`entity: personal_schedule`） |
| Agent ChangeSet 创建目标日程 | 是（`entity: schedule`） |

## 9. Agent 日程安排能力（v0.2.0 补充）

### 9.1 背景

用户除手动在日历上添加/修改个人日程外，也应能通过与小律对话完成同样的事。当前 Agent 的 `propose_change_set` 仅笼统使用 `entity: "schedule"`，容易把「团队周会占位」误判为「关联某个目标的任务安排」，或把「每天练口语」误建成单次日程块而非 Routine。

需要在 Agent 层建立**三类日程意图**的清晰边界，并在 ChangeSet、上下文读取、意图路由与 UI 预览中保持一致。

### 9.2 三类日程意图定义

| 意图类型 | ChangeSet entity | 用户典型表述 | 数据特征 | 禁止混用 |
|---------|------------------|-------------|---------|---------|
| **个人日程** | `personal_schedule` | 「下午有个会」「把 12–13 点标成午休」「明天上午没空」 | 仅 title + 时间；`goalId`/`taskId`/`routineId` 均为空 | 不得关联目标、任务、Routine；不得带 `recurrenceRule` |
| **目标日程** | `schedule` | 「把数据库设计排到明天下午」「给这个任务安排 2 小时」 | title + 时间 + `goalId`（必填）+ 可选 `taskId`/`taskIds` | 不得带 `recurrenceRule`；不得用于纯占位 |
| **Routine 规则** | `routine` | 「每天早上 9 点练口语 20 分钟」「每周二四力量训练」 | title + `recurrenceRule` + `goalId` + `durationMinutes` 等 | **不得**用 `schedule` 批量生成重复块；日历实例由系统按规则展开 |

**判定优先级（Agent 内部）：**

1. 出现重复语义（每天/每周/工作日）→ `routine`
2. 出现占位/外部占用语义，且未指向具体目标或任务 → `personal_schedule`
3. 出现推进目标/任务语义，或页面已选中 goal/task → `schedule`
4. 仍无法区分 → 追问用户：「这是时间占位，还是要推进某个目标？」

### 9.3 意图识别信号

**个人日程（personal_schedule）关键词：**

会议、周会、例会、面试、约会、聚餐、通勤、午休、睡觉、健身课（外部）、没空、占用、占位、阻塞、个人日程、日历上标一下、飞书/Outlook 上的事

**目标日程（schedule）关键词：**

安排任务、排到、推进、做 xxx、投入、专注时间、把「{任务名}」放到、关联目标

**Routine 关键词：**

每天、每周、工作日、周末、重复、习惯、长期坚持、Routine

**页面上下文加权：**

| 当前页面 | 默认倾向 |
|---------|---------|
| 今日安排（today） | 个人日程与目标日程均可能；无目标选中时，占位类表述优先 `personal_schedule` |
| 任务详情（task-detail） | 默认 `schedule`，并带上当前 taskId |
| 目标详情（goal-detail） | 默认 `schedule` 或 `routine`（视重复语义） |
| Routine 页（routines） | 默认 `routine` 调整，而非单次 schedule |

### 9.4 ChangeSet 操作规范

#### 创建个人日程

```json
{
  "type": "create",
  "entity": "personal_schedule",
  "payload": {
    "title": "团队周会",
    "date": "2026-06-26",
    "start": "14:00",
    "end": "15:00"
  }
}
```

或使用 ISO 时间：

```json
{
  "title": "午休",
  "startsAt": "2026-06-26T04:00:00.000Z",
  "endsAt": "2026-06-26T05:00:00.000Z"
}
```

**约束：** payload 中不得出现 `goalId`、`taskId`、`routineId`、`recurrenceRule`。也支持 `scheduleKind: "personal"` 写在 `entity: "schedule"` 上，服务端会规范化为 `personal_schedule`。

#### 创建目标日程

```json
{
  "type": "create",
  "entity": "schedule",
  "payload": {
    "title": "梳理数据库实体关系",
    "goalId": "{goalId}",
    "taskId": "{taskId}",
    "date": "2026-06-26",
    "start": "14:00",
    "end": "16:00"
  }
}
```

**约束：** 必须能解析出 `goalId`（直接提供，或通过 `taskId` 反查）。不得带 `recurrenceRule`。

#### 修改 / 删除个人日程

```json
{
  "type": "update",
  "entity": "personal_schedule",
  "entityId": "{scheduleBlockId}",
  "before": {},
  "after": { "title": "产品评审会", "start": "15:00", "end": "16:00", "date": "2026-06-26" }
}
```

```json
{
  "type": "archive",
  "entity": "personal_schedule",
  "entityId": "{scheduleBlockId}",
  "before": {}
}
```

修改目标日程仍用 `entity: "schedule"`。若将个人日程改为关联目标，应拆成 archive + create schedule，而非在同一操作上混写字段。

### 9.5 Agent 工作流

```
用户表达日历变更意图
    ↓
read_schedule_window（确认时间冲突与已有块类型）
    ↓
[可选] read_goal_context（仅当涉及目标/任务/Routine 时）
    ↓
判断意图 → personal_schedule | schedule | routine
    ↓
propose_change_set（写入对应 entity）
    ↓
用户确认 → 落库
```

**小律在回复中应说明：**

- 识别到的日程类型（个人占位 / 目标推进 / Routine 规则）
- 若与现有安排冲突，提示冲突块
- 个人日程不会计入目标投入

### 9.6 上下文增强

`read_schedule_window` 返回的每条日程块增加 `blockKind` 字段：

| blockKind | 含义 |
|-----------|------|
| `personal` | 个人日程占位 |
| `goal_task` | 关联目标/任务的一次性安排 |
| `routine_occurrence` | Routine 展开实例（`source: routine_occurrence`） |

Agent 调整已有块时，必须先读取 `blockKind`，对个人块使用 `personal_schedule` entity 进行修改。

### 9.7 服务端校验

| 操作 | 校验规则 |
|------|---------|
| create personal_schedule | 拒绝 payload 中含 goalId/taskId/routineId/recurrenceRule |
| create schedule | 要求 goalId 或有效 taskId；拒绝 recurrenceRule |
| update personal_schedule | 目标块 blockKind 必须为 personal；after 不得写入关联字段 |
| update schedule | 目标块 blockKind 为 goal_task；可改关联与时间 |

### 9.8 Agent 验收标准

- [ ] 用户说「明天下午 3 点有个会」，Agent 生成 `personal_schedule` 草案
- [ ] 用户说「把实现手动编辑流程排到周四下午」，Agent 生成带 goalId/taskId 的 `schedule` 草案
- [ ] 用户说「每天早上 9 点练口语」，Agent 生成 `routine` 而非多个 schedule
- [ ] 用户说「把周会改到 4 点」，Agent 先读 schedule_window，对个人块用 `personal_schedule` update
- [ ] ChangeSet 预览中个人日程显示为「个人日程」类型
- [ ] 意图模糊时 Agent 追问，不擅自关联目标
- [ ] 本地模式 applyLocalChangeSet 与数据库模式行为一致

## 10. 验收标准（手动 + Agent）

**手动交互：**

- [ ] 今日安排页双击空白时段，出现「个人日程 / 目标日程」选择
- [ ] 个人日程仅需填写名称与时间即可保存
- [ ] 个人日程在日视图以独立样式展示，标签为「个人」
- [ ] 个人日程可拖动、拉伸、编辑、删除
- [ ] 个人日程不显示「记录执行」按钮
- [ ] 个人日程不关联目标时，任务状态不受影响
- [ ] 从目标/任务页安排日程时，不出现类型选择，行为与现版一致
- [ ] 数据库模式与本地模式行为一致

**Agent 交互：** 见 §9.8。
