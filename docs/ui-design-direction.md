# Rhythm & Routine UI 设计方向

## 当前决策

MVP 默认采用 **Soft Humanist** 视觉风格。

决策日期：2026-06-19

这个方向应该让产品感觉像一个理解用户节奏、会陪伴用户推进目标的日常伙伴，而不是冷静、凌厉的任务管理工具。

## 选择原因

- 有陪伴感，符合“小律”的 AI 伙伴定位。
- 视觉压力较低，适合每天反复使用。
- 可以承载目标、日程、回顾等严肃信息，又不会显得像企业后台。
- 与产品“根据人的状态调整计划”的核心价值一致。

## 视觉特征

- 使用低饱和、偏暖的背景色，可搭配淡紫、柔绿和浅珊瑚色。
- 卡片采用柔和圆角、轻微层次和克制阴影，避免强烈的玻璃拟态。
- 排版清晰但不过度几何化，标题语气尽量自然。
- 状态文案使用支持性表达，例如“今天已经推进了不少”，避免过度督促。
- 动效应轻快、短促，主要用于响应、完成反馈和页面层级变化。

## 实现时的边界

Soft Humanist 是设计方向，不是不可变动的模板。开发过程中可根据真实内容密度、可用性、无障碍要求和移动端效果调整颜色、间距、圆角、阴影和局部布局。

需要优先保持的是：

1. 低压力的整体气质。
2. 清晰的信息层级。
3. 自然、支持性的产品语气。
4. “小律”作为陪伴者，而不是命令者的感受。

## 未来主题能力

后续可考虑允许用户选择不同风格，但不纳入 MVP 首版范围。

建议从开发初期就把以下内容抽象为 design tokens，避免将视觉值写死在组件中：

- 背景色、表面色、文字色和功能色。
- 字体角色与字号层级。
- 间距、圆角、边框和阴影等级。
- 动效时长和 easing。
- 各类状态的语义颜色。

候选主题可保留 Calm Swiss、Editorial Ritual 和 Night Focus，但所有主题都应共用相同的信息架构、交互规则与无障碍标准。
