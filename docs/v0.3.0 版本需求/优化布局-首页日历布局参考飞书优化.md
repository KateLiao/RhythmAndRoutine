# 需求：将 Rhythm & Routine 日历视图改造成类似飞书日历的高信息密度设计

## 背景

当前 Rhythm & Routine 的日历视图在视觉上比较“卡片化”，每个日程块高度较大，留白较多，导致单位屏幕内可呈现的事务数量较少。相比之下，飞书日历在「日 / 周 / 月」三种视图下的信息呈现效率更高，尤其是：

1. 日视图和周视图共用同一套「纵向滚动时间轴」设计。
2. 时间块高度与真实时长严格对应，视觉上更紧凑。
3. 日程块信息密度更高，不需要过大的卡片容器。
4. 月视图以网格形式展示事件摘要，适合快速查看整月分布。
5. 顶部日期导航、视图切换、今天按钮位置清晰。

因此，需要将 Rhythm & Routine 当前日历视图改造成更接近飞书日历的信息架构和交互方式，同时保留 RnR 的视觉风格、颜色体系和 Rhythm / Routine 的产品特性。

---

## 目标

将 RnR 的日历模块从当前偏「卡片式日程展示」升级为更高效的「专业日历视图」。

核心目标：

1. 提高日程信息呈现密度。
2. 日视图和周视图使用统一的滚动时间轴。
3. 月视图使用标准月历网格。
4. 日程块高度根据时间长度自动计算。
5. 保留当前时间红线。
6. 保留日程块与 Task / Routine / Goal 的关联信息。
7. 保留 RnR 的柔和紫色视觉风格，但布局结构向飞书日历靠拢。

---

# 一、整体布局调整

## 1.1 顶部工具栏

日历区域顶部需要包含：

- 今天按钮
- 上一页 / 下一页按钮
- 当前日期 / 当前月份标题
- 日 / 周 / 月视图切换器

参考飞书日历结构：

```text
[今天] [<] [>]  2026年7月9日 今天                         [日] [周] [月]
```

不同视图下标题显示规则：

### 日视图

```text
2026年7月9日 今天
```

如果不是今天：

```text
2026年7月10日 周五
```

### 周视图

```text
2026年7月
```

或：

```text
2026年7月5日 - 7月11日
```

优先使用简洁的：

```text
2026年7月
```

### 月视图

```text
2026年7月
```

## 1.2 视图切换器

右上角使用 segmented control：

```text
[ 日 ] [ 周 ] [ 月 ]
```

当前选中的视图使用浅紫色背景和紫色文字。

要求：

* 三个视图切换时不跳出当前页面。
* 切换视图后，当前选中的日期应该被保留。
* 例如在 7 月 9 日日视图切换到周视图，应展示包含 7 月 9 日的那一周。
* 从周视图切换到月视图，应展示 2026 年 7 月。

# 二、日视图设计

## 2.1 日视图核心结构

日视图使用纵向滚动时间轴。

结构：

```text
顶部工具栏
日期 / 时区头部
纵向时间轴
日程块
当前时间红线
```

## 2.2 时间轴

左侧固定显示时间刻度：

```text
GMT+8
11:00
12:00
13:00
14:00
15:00
...
23:00
```

要求：

* 时间刻度在左侧单独一列。
* 每小时一条横向分割线。
* 时间轴支持纵向滚动。
* 时间范围建议默认展示 08:00 - 24:00。
* 如果后续需要，可以支持配置工作时间范围。
* 当前视图滚动初始位置应尽量靠近当前时间附近，而不是默认从 00:00 开始。

## 2.3 日程块高度

日程块高度必须根据实际时长计算，而不是固定大卡片高度。

建议规则：

```text
1 小时 = 60px 或 72px
30 分钟 = 30px 或 36px
15 分钟 = 15px 或 18px
```

可以根据实际 UI 调整，但必须满足：

* 15 分钟日程块也能显示。
* 2 小时日程块视觉上明显比 30 分钟日程块更高。
* 日程块的 top 位置由 start_time 决定。
* 日程块的 height 由 end_time - start_time 决定。

示例：

```ts
const HOUR_HEIGHT = 72;

const top = minutesFromDayStart(startTime) / 60 * HOUR_HEIGHT;
const height = durationMinutes / 60 * HOUR_HEIGHT;
```

## 2.4 日程块样式

日程块需要更加紧凑，类似飞书日历，而不是当前 RnR 的大卡片。

日程块内部建议展示：

```text
任务标题
开始时间 - 结束时间
```

或者更紧凑：

```text
17:30 test2
17:30 - 19:30
```

RnR 可增加轻量标签，但不能占用太多高度。

建议结构：

```text
[小色条] 17:30 产品原型设计
        17:30 - 19:30 · 深度工作
```

## 2.5 日程块颜色

根据对象类型区分颜色：

```text
Task：浅紫 / 蓝紫
Routine：浅绿
Review：浅黄 / 米色
Break / Recovery：浅灰绿
Goal-related deep work：紫色强调
```

要求：

* 保持柔和低饱和。
* 不要使用大面积高饱和色。
* 日程块左侧可使用 3-4px 色条增强识别。
* 当前进行中的日程块可以有更明显边框。

## 2.6 当前时间线

保留红色当前时间线。

要求：

```text
红色圆点 + 红色水平线 + 当前时间标签
```

效果类似飞书：

```text
17:53  —————————————————————————
```

具体要求：

* 当前时间线只在今天的日视图和今天所在周的周视图中显示。
* 当前时间线横跨日程区域。
* 左侧时间刻度处显示当前时间。
* 在线条起点处显示红色圆点。
* 右侧可选显示小标签，例如“现在 17:53”。

# 三、周视图设计

## 3.1 周视图核心结构

周视图和日视图共用同一套纵向滚动时间轴。

结构：

```text
顶部工具栏
一周日期头部
左侧时间轴
7 列日期网格
日程块
当前时间红线
```

## 3.2 日期头部

顶部横向展示一周 7 天：

```text
周日 5
周一 6
周二 7
周三 8
周四 9
周五 10
周六 11
```

当前选中日期高亮，例如：

```text
周四 9
```

高亮规则：

* 日期数字使用品牌紫 / 蓝色。
* 当前日期列可以有轻微背景高亮。
* 今天需要特别标识。

## 3.3 周视图网格

要求：

* 左侧保留时间刻度。
* 右侧为 7 列，每列代表一天。
* 每小时有横向分割线。
* 每天之间有竖向分割线。
* 整体支持纵向滚动。
* 日程块放置在对应日期列内。
* 日程块 top 和 height 同日视图一样，根据时间计算。

## 3.4 周视图日程块

周视图由于每列宽度较窄，日程块内容需要更简洁。

建议展示：

```text
标题
时间
```

如果空间不足，只展示：

```text
17:30 test2
```

hover 或点击后再展示详情。

要求：

* 同一时间段内重叠日程需要并排或缩窄显示。
* 如果没有重叠，则占满当前日期列宽度。
* 日程块样式与日视图一致，但字号略小。
* 支持点击查看 / 编辑日程详情。

## 3.5 当前时间线

周视图中的当前时间线应横跨整个周网格。

要求：

* 如果当前周包含今天，则显示红色当前时间线。
* 当前时间线穿过所有 7 天列。
* 在今天对应列的左侧或线起点处显示红点。
* 左侧时间轴显示当前时间。

---

# 四、月视图设计

## 4.1 月视图核心结构

月视图使用标准月历网格。

结构：

```text
顶部工具栏
周标题行
月份日期网格
每日事件摘要
```

## 4.2 周标题

顶部展示：

```text
周日 周一 周二 周三 周四 周五 周六
```

## 4.3 日期网格

要求：

* 每一行代表一周。
* 每一列代表一天。
* 显示当前月完整日期。
* 上月 / 下月补位日期使用浅灰色。
* 今天或当前选中日期高亮。
* 当前选中日期单元格可以有淡灰 / 淡紫背景。

## 4.4 月视图事件展示

月视图不展示时间轴，只展示当天事件摘要。

每个事件建议展示：

```text
小色点 + 时间 + 标题
```

示例：

```text
17:00 测试
17:30 test2
```

要求：

* 每个日期格最多展示 2-3 条事件。
* 超出后展示：

```text
+3 更多
```

* 点击日期进入日视图。
* 点击事件打开事件详情。
* 事件颜色与对象类型保持一致。

## 4.5 今日高亮

今日日期显示为品牌色圆形或胶囊样式。

示例：

```text
9
```

使用紫色 / 蓝紫色圆形背景，白色数字。

当前选中日期可使用浅紫背景。

# 五、日程块数据映射

当前 RnR 的 Schedule Block 需要映射到日历视图。

Schedule Block 至少需要包含：

```ts
type ScheduleBlock = {
  id: string;
  title: string;
  startTime: string; // ISO datetime
  endTime: string;   // ISO datetime
  type: "task" | "routine" | "review" | "break" | "other";
  status: "scheduled" | "in_progress" | "completed" | "skipped" | "rescheduled" | "cancelled";
  linkedGoalId?: string;
  linkedTaskId?: string;
  linkedRoutineId?: string;
  rhythmTag?: "deep_work" | "light_task" | "body" | "review" | "recovery";
};
```

日历展示时：

```text
title -> 日程块主标题
startTime / endTime -> 时间位置和高度
type / rhythmTag -> 颜色与图标
status -> 状态图标或透明度
```

## 5.1 状态展示

建议规则：

```text
scheduled：正常显示
in_progress：高亮边框 / 当前进行中
completed：显示勾选图标，透明度略降低
skipped：虚线边框或灰色
rescheduled：显示改期标识
cancelled：灰色 + 删除线，可选隐藏
```

# 六、交互要求

## 6.1 点击日程块

点击日程块后打开详情面板或弹窗。

详情内容包括：

* 标题
* 时间
* 类型
* 关联目标 / 任务 / Routine
* 当前状态
* 记录执行
* 编辑
* 删除 / 取消
* 改期

## 6.2 拖拽调整时间

如果当前已有拖拽能力，需要适配新时间轴。

要求：

* 拖动日程块可改变 startTime / endTime。
* 在日视图中上下拖动改变时间。
* 在周视图中左右拖动改变日期，上下拖动改变时间。
* 月视图中拖动事件到另一天可改变日期，但不改变时间。
* 拖拽结束后更新 Schedule Block。
* 如果日程关联 Routine，需要询问是否只调整本次实例。

## 6.3 拉伸调整时长

日视图 / 周视图支持拖拽日程块底部改变时长。

要求：

* 最小粒度 15 分钟。
* 拉伸后更新 endTime。
* 显示实时辅助线或时间提示。
* 结束后保存。

## 6.4 新建日程

用户可以通过：

1. 点击空白时间格。
2. 拖拽选择一段时间。
3. 点击「安排事情」按钮。

来新建日程。

在日 / 周视图中：

* 点击某个时间位置，默认创建 30 分钟日程。
* 拖拽选择时间段，按选中范围创建日程。

在月视图中：

* 点击某一天，打开新建日程弹窗。
* 默认日期为选中日期。
* 默认时间可以为空或使用用户偏好时间。

# 七、视觉要求

## 7.1 保留 RnR 品牌风格

虽然结构参考飞书日历，但视觉上仍然要保持 Rhythm & Routine 的风格。

保留：

* 柔和紫色主色
* 大圆角页面容器
* 轻微阴影
* 浅色背景
* 低饱和色块
* 清爽留白
* AI-native 的轻盈感

不要完全复制飞书的蓝色和企业工具感。

## 7.2 提高信息密度

相比当前 RnR 日历视图，需要减少：

* 过大的日程块高度
* 过大的日历外层留白
* 过多装饰性文本
* 重复的日期信息
* 非必要卡片包裹

日程区域应该更像一个高效日历，而不是多个大卡片堆叠。

## 7.3 建议尺寸

```ts
const TIME_COLUMN_WIDTH = 64;
const HOUR_HEIGHT = 72;
const MIN_EVENT_HEIGHT = 18;
const DAY_HEADER_HEIGHT = 72;
const TOOLBAR_HEIGHT = 64;
```

可以根据实际布局微调。

# 八、响应式与容器适配

## 8.1 首页嵌入场景

当前 RnR 首页中，日历视图可能嵌入在主内容卡片内。

要求：

* 日历组件需要支持在固定高度容器内滚动。
* 不要让整个页面滚动失控。
* 日 / 周视图内部时间轴区域可以滚动。
* 工具栏和日期头部固定在顶部。
* 时间轴内容滚动时，顶部日期头部保持 sticky。

## 8.2 宽度适配

日视图：

* 单日列占满可用宽度。

周视图：

* 7 列等宽。
* 窄屏时允许横向滚动，或降低文字密度。

月视图：

* 7 列等宽。
* 每个日期格高度一致。

# 九、验收标准

## 9.1 日视图验收

* 可以显示单日纵向时间轴。
* 日程块位置与开始时间一致。
* 日程块高度与持续时间一致。
* 当前时间红线显示正确。
* 可以点击日程块查看详情。
* 日程块比当前版本更紧凑，屏幕内可展示更多事务。

## 9.2 周视图验收

* 周视图使用与日视图一致的纵向时间轴。
* 顶部展示 7 天日期。
* 日程块出现在对应日期列和时间位置。
* 当前时间线在当前周显示。
* 日程块支持点击查看详情。

## 9.3 月视图验收

* 月视图使用标准 7 列月历网格。
* 当前月日期、上月/下月补位日期展示正确。
* 今天高亮。
* 每天能展示事件摘要。
* 超出事件数量时显示“+N 更多”。

## 9.4 视图切换验收

* 日 / 周 / 月切换正常。
* 切换视图时保留当前日期上下文。

# 十一、最终效果描述

改造后的 RnR 日历视图应该具备以下体验：

```text
日视图：
像专业日历一样清楚地看到一天中每个时间段被什么事情占用。

周视图：
用同一套时间轴快速比较一周内不同日期的安排密度。

月视图：
快速看到整月任务、Routine 和重要安排的分布。

整体：
保留 Rhythm & Routine 的柔和节奏感，但在日程呈现效率上接近飞书日历。
```
